# CapitalFlow LMS â€” Configuration

## Principles

- **No hardcoded credentials** anywhere in code â€” all secrets in AWS Secrets Manager
- **No hardcoded infrastructure references** â€” all resource ARNs/URLs injected as environment variables by CDK at deploy time
- **Frontend config** is build-time only (Vite `.env` files); never include secrets
- **Backend config** uses `process.env` for non-secret config; `getSecret(arn)` helper for secrets

| Variable | Description | Example |

## Secrets in AWS Secrets Manager

These must be created manually after initial CDK deploy. All other secrets are auto-generated by CDK.

| Secret Path | Format | Notes |
|-------------|--------|-------|
| `capitalflow/{env}/anthropic-api-key` | `{ "key": "sk-ant-..." }` | Create at `console.aws.amazon.com/secretsmanager` |

Auto-generated by CDK (no manual action needed):
- `capitalflow/{env}/db-credentials` â€” Aurora master password
- `capitalflow/{env}/jwt-secret` â€” 256-bit random secret for JWT signing

## Frontend Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|

| `VITE_COGNITO_CLIENT_ID` | Cognito app client ID | `1a2b3c4d5e...` |
| `VITE_COGNITO_REGION` | AWS region | `us-east-1` |
| `VITE_APP_ENV` | App environment label | `dev` \| `prod` |

**Files:**
- `frontend/.env.example` â€” template checked into source control
- `frontend/.env.local` â€” local dev overrides, gitignored
- `frontend/.env.production` â€” production values, populated by CI/CD pipeline, gitignored

## Local Development

For local development, backend uses a local PostgreSQL 16 instance:

```
# backend/.env.local (gitignored)
NODE_ENV=development
DATABASE_URL=postgresql://postgres:password@localhost:5432/capitalflow_dev
JWT_SECRET=local-dev-secret-not-for-production
ANTHROPIC_API_KEY=sk-ant-...
DOCUMENTS_BUCKET=capitalflow-documents-dev
```

Frontend local dev:
```
# frontend/.env.local (gitignored)
VITE_API_URL=http://localhost:3001
VITE_APP_ENV=dev
```
